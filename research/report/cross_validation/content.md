## YOLO Dataset Pipeline

- Objective: Prepare a species-labeled YOLO dataset, run segmentation with a train/test split only, materialize strain-held-out species classification folds, generate colony-crop datasets, preview augmentation policy visually, and export classifier-free backbone weights for retrieval-style feature extraction.
- Dataset version: `Dataset/manual_labeled_data_roboflow/` relabeled via `Dataset/strain_to_specy.csv`
- Segmentation split rule: deterministic random train/test split with no separate validation partition
- Cross-validation split rule: one selected held-out strain per species per fold, with no separate validation partition and no reuse of Roboflow split membership for fold assignment
- Crop dataset strategy: derive colony-centric crops from YOLO labels, preserve parent train/test assignment, and record crop provenance in `crop_assignment.csv`
- Augmentation debug surface: render preview grids from one source image or crop before long training runs
- Weight export contract: save backbone-only weights under `weights/yolo_finetuned/` so `src/experiments/feature_extraction/feature_extractors.py` continues to load classifier-free embeddings
- Fine-tuning command surface: `python -m src.experiments.finetune_dl.train_yolo_crops --dataset-root <dataset-root> --model-name <ResNet50|MobileNetV2|EfficientNetB1>`
- Segmentation command surface: `python -m src.experiments.yolo_segmentation.run --dataset-root <dataset-root> --epochs <N>`
- Output roots:
  - Derived dataset: `Dataset/manual_labeled_data_roboflow_species/`
  - Fold datasets: `Dataset/manual_labeled_data_roboflow_species_cv/fold_*`
  - Crop datasets: `Dataset/manual_labeled_data_roboflow_species_crops/` or fold-local `*_crops`
  - Cross-validation results: `results/cross_validation_yolo/`
  - Segmentation metadata: `results/cross_validation_yolo/segmentation/`
- Key metrics:
  - preparation processed/skipped/failed counts
  - segmentation train/test sample counts and metrics summary
  - cross-validation fold counts, held-out strain assignments, aggregate CSV summaries, and figure inventory
  - crop dataset train/test counts and crop assignment provenance
  - fine-tuning history paths and exported weight paths
- Figures:
  - fold comparison plots under `results/cross_validation_yolo/`
  - augmentation preview grids under `results/augmentation_preview/`
  - any segmentation summary figures generated by the YOLO segmentation workflow
