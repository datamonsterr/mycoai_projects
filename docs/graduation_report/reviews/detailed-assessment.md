# Detailed Assessment

## Chapter 1

### Problems
- Problem statement still too soft and descriptive.
- System-side pain is under-specified.
- Proposed-solution wording contains grammar issues and weak precision.
- Scope section is too short for thesis expectations.

### Required fixes
- Define domain terms once, then move to explicit two-problem framing.
- State why single-image classification is insufficient for strain-level identification.
- State why current lab workflow needs governed retrieval + curation + indexing support.
- Split scope into research scope and system scope.

## Chapter 2

### Problems
- Methodology and result language still mixed.
- Several experiment subsections are too short.
- Aggregation and thresholding are under-explained compared with their importance.
- Figures are often shown without strong interpretation.
- Code-style strategy names leak into thesis prose.

### Required fixes
- Add one dominant end-to-end IO visualization for full pipeline.
- Keep methodology captions descriptive, not evaluative.
- Expand each experiment subsection with objective, setup, results, interpretation, implication.
- Replace raw strategy identifiers with academic names in main prose.
- Add stronger threshold-method summary in main chapter.

## Chapter 3

### Problems
- Functional scope still too informal.
- Screenshot count too high relative to argument value.
- Training/model-promotion claims exceed implementation evidence.

### Required fixes
- Distinguish implemented workflow from prototype workflow.
- Reframe training lifecycle as partial scaffold unless implementation is completed.
- Keep retrieval/indexing/governance claims backed by code/tests.
- Reduce screenshots and favor composite workflow + architecture + ERD + one or two evidence screenshots.

## Chapter 4

### Problems
- Weak measurable evaluation.
- Limited tie-back to research and product outputs.

### Required fixes
- Add measurable evidence table for agentic workflow impact.
- Connect agent workflows to experiment throughput, automation, and report generation.

## Chapter 5

### Problems
- Summary too generic.
- Future work too long and product-pitch styled.
- Segmentation statement inconsistent with YOLO-adoption story.

### Required fixes
- Summarize actual validated findings.
- State threshold limitation honestly.
- Align future work with current implementation and research gaps.
