\section{Motivation}\label{motivation-agentic}

Modern software development is often bottlenecked by the manual cycle of ``hypothesis \(\rightarrow\) implementation \(\rightarrow\) testing''. Agentic Engineering seeks to automate this loop by using LLM-based agents that can write code, run tests, and analyze results autonomously. This approach draws from Karpathy's autoresearch concept~\cite{karpathy2026autoresearch}, which frames AI-driven code modification as an iterative keep/discard loop, and from Optuna's define-by-run hyperparameter optimisation paradigm~\cite{akiba2019optuna}.

\section{Multi-Agent Architecture}\label{multi-agent-architecture}

The project implemented the \textbf{Autolab} system (documented in the project monorepo under \texttt{.opencode/agents/}), a five-agent orchestration layer inspired by Karpathy's autoresearch concept~\cite{karpathy2026autoresearch}: 1. \textbf{Autolab (Orchestrator)}: Delegates tasks to subagents. 2. \textbf{Researcher}: Scouts literature and suggests new hypotheses. 3. \textbf{Planner}: Manages the experiment queue and assigns IDs. 4. \textbf{Worker}: Implements the hypothesis in an isolated git worktree and runs the experiment. 5. \textbf{Reporter}: Summarizes the results and identifies the new ``Best F1'' score.

\section{Reducing Hallucinations}\label{reducing-hallucinations}

To ensure the AI agents produced reliable code, three guardrails were implemented: - \textbf{Spec-Driven Development (SDD)}: Using \texttt{Speckit}~\cite{speckit}, an agent skills framework for spec-driven workflows, combined with \texttt{OpenSpec}~\cite{openspec}, a CLI tool for Living Specification management. Agents must follow the spec, and any drift is detected and resolved through automated analysis. - \textbf{Test-Driven Development (TDD)}: Agents are required to write a failing test before implementing a fix, ensuring that every change is verified. - \textbf{Specialized Skills}: Instead of relying on general knowledge, agents are provided with ``Skills'' (e.g., \texttt{vercel-react-best-practices}) which act as bounded, high-quality instructions.

\section{Overcoming Context Window Limits}\label{overcoming-context-window-limits}

As the codebase grew, managing the LLM's context window became critical:

\subsection{Caveman Mode}\label{caveman-mode}

A token-compression communication style that removes fluff and articles, reducing input size by approximately 75\% without losing technical substance.

\subsection{Subagents and Worktrees}\label{subagents-and-worktrees}

Using isolated git worktrees and a routing layer to direct queries to the most appropriate subagent, ensuring each agent only receives context relevant to its specific task.

\subsection{Reusable Skills}\label{reusable-skills}

Encoding complex workflows (e.g., \texttt{vercel-react-best-practices}, \texttt{fastapi}) into bounded, high-quality skill definitions that constrain agent behavior to validated patterns.

\section{End-to-End Agent-Browser Testing}\label{e2e-agent-browser}

To validate the complete retrieval pipeline against research experiment
configuration, an end-to-end test was conducted using \texttt{agent-browser},
a headless browser automation CLI built in Rust with Chrome DevTools Protocol
(CDP). The test simulates a regular user performing the full species retrieval
workflow.

\subsection{Test Configuration}

The E2E test validates the same configuration used in research experiments:

\begin{itemize}[nosep]
  \item \textbf{Feature extractor}: EfficientNetB1\_finetuned (1,280-dim vectors)
  \item \textbf{Collection}: myco\_fungi\_features\_full\_finetuned
  \item \textbf{K}: 11 nearest neighbors
  \item \textbf{Environment strategy}: same\_media (E1, query only same growth medium)
  \item \textbf{Aggregation}: freq\_strength
  \item \textbf{Threshold formula in current webapp}: gnorm\_0\_2 (backend default at time of writing; research-best formula after regeneration is gap\_0\_2)
\end{itemize}

\subsection{Test Flow}

\begin{enumerate}[nosep]
  \item \textbf{Login}: Test user authenticates through the sign-in page.
  \item \textbf{Dashboard}: Verify dataset overview with 727 images, 136 species.
  \item \textbf{Upload}: Navigate to Retrieve Species page, showing batch upload UI.
  \item \textbf{Segmentation}: Review K-means segmented colonies with editable bounding boxes.
  \item \textbf{Results}: View ranked species predictions with threshold confidence badge.
\end{enumerate}
