# Common Mistakes to Avoid

- Using itemize where ordered procedure or ranked scope needs enumerate.
- Using raw code names like `freq_strength` in prose without human-readable explanation.
- Inserting figures before explaining what question they answer.
- Writing captions that only repeat figure content without stating why it matters.
- Showing charts with undefined acronyms, symbols, or environment codes.
- Leaving experiment subsections as one paragraph plus one figure.
- Treating prototype workflows as deployed workflows.
- Writing future work that contradicts current chosen architecture.
- Keeping long dense paragraphs where numbered structure is clearer.
- Referring to filepaths in thesis body instead of conceptual modules.
- Manually editing generated assets instead of fixing generator source.
